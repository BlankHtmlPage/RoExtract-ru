Усталюйце [Microsoft Visual C++ Redistributable](https://learn.microsoft.com/ru-ru/cpp/windows/latest-supported-vc-redist) (артыкул на рускай мове)

### Ці ўплывае гэта на кліент Roblox?
Не, гэты інструмент адкрывае файлы, якія ваш кліент ужо стварыў. Вы можаце паглядзець іх самі ў %Temp%\Roblox

### Гэта вірус?
Не, гэта не вірус. Як і ў любога іншага свабоднага ПЗ з адкрытым зыходным кодам, наш код адкрыты для прагляду кім заўгодна, таму схаваць у ім нешта шкоднаснае немагчыма. Зборкі таксама бяспечныя, бо зараз яны ствараюцца аўтаматычна праз GitHub Actions - гэта гарантуе поўную празрыстасць. Калі вы ўсё роўна не давяраеце гэтаму ПЗ, вы можаце скарыстацца вэб-дэма па спасылцы https://aeen123.github.io/RoExtract/demo (для гэтага нічога спампоўваць не трэба)

Гэта акно з'яўляецца, калі Windows выяўляе праграму ад невядомага выдаўца. У такім выпадку проста націсніце "Падрабязней", а затым - "Выканаць у любым выпадку".

### Гэта можа выклікаць бан?
Не, у адрозненне ад чытоў, гэта ПЗ **не** ўзаемадзейнічае з Roblox, робячы гэтую праграму бяспечным спосабам здабывання рэсурсаў.

### Мае здабытыя рэсурсы не прайграюцца ў медыя-плэеры, што рабіць?
Некаторыя медыя-плэеры могуць не падтрымліваць фармат здабытага файла. У такім выпадку паспрабуйце іншы прайгравальнік, які падтрымлівае ўсе фарматы, напрыклад VLC. **Калі файл сапраўды пашкоджаны, калі ласка, [паведаміце пра памылку.](https://github.com/AeEn123/RoExtract/issues)**

### Чаму файлы KTX знаходзяцца ў іншай укладцы? Ці не павінны яны знаходзіцца ва ўкладцы Тэкстуры?
Фармальна так, павінны, але большасць праграм для прагляду выяў не падтрымліваюць файлы KTX, таму лепш перанесці іх у іншую ўкладку, каб пазбегнуць праблем з сумяшчальнасцю. Гэта ўкладка прызначана толькі для дасведчаных карыстальнікаў.

### Чаму файлы RBXM адлюстроўваюцца проста як "Instance" у Roblox Studio?
Roblox Studio не падтрымлівае кэшаваныя файлы RBXM. Гэтыя файлы могуць утрымліваць даныя з гульняў, але дэталёва мы пакуль гэтага не вывучалі.

### Ці займае гэта месца на дыску з часам?
Ваш кэш Roblox сапраўды займае ўсё больш месца з часам, але сама праграма не павялічвае выкарыстанне дыска, калі толькі вы не здабываеце вялікую колькасць файлаў (якія можна лёгка выдаліць).


# Выкарыстанне
## Укладкі
Вы бачыце некалькі ўкладак. RoExtract размяркоўвае файлы па катэгорыях. Вы можаце адфільтраваць іх, націснуўшы на ўкладку.
## Панэль інструментаў
Кожная кнопка ў панэлі інструментаў дазваляе вам рабіць розныя аперацыі з дырэкторыяй або рэсурсам, вы таксама можаце адкрыць гэта меню, націснуўшы ПКМ. Вы можаце выключыць панэль інструментаў уверсе экрана ў наладах, опцыя **Уключыць панэль інструментаў** у раздзеле **Паводзіны**.
## Навігацыя з дапамогай клавіятуры і гарачыя клавішы
Праграма распрацавана так, каб яе было зручна выкарыстоўваць мышшу, але пры гэтым падтрымлівае навігацыю з дапамогай клавіятуры для дасведчаных карыстальнікаў. Спалучэнні клавіш адлюстроўваюцца прама на кнопках, каб вы маглі хутка іх убачыць.

Вы можаце мяняць укладкі з дапамогай Alt (або Ctrl) + 1-8. Вы можаце выбіраць рэсурсы з дапамогай Tab і распакоўваць/адкрываць з дапамогай Enter.
## Меню налад
У меню налад вы знойдзеце агульныя параметры кастамізацыі і таксама выбар дзеянняў з кэшам. Тут вы можаце распакаваць увесь кэш, змяніць дырэкторыю або ачысціць кэш.

# Усталяванне на Linux
## Flatpak (ЭКСПЕРЫМЕНТАЛЬНА)
> [!WARNING]
> Падтрымка Flatpak ЭКСПЕРЫМЕНТАЛЬНАЯ, выкарыстоўвайце на свой страх і рызыку.

На дадзены момант гатовых пакетаў Flatpak няма. Выконвайце інструкцыі ў [кіраўніцтве](../../packages/flatpak/README_ru-RU.md), каб сабраць пакет самастойна.

## Arch Linux
Вы можаце ўсталяваць RoExtract у Arch Linux, выкарыстоўваючы файл `PKGBUILD`, які знаходзіцца ў `packages/arch`
Прыклад усталявальнага скрыпта:
```bash
mkdir /tmp/RoExtract
cd /tmp/RoExtract
wget raw.githubusercontent.com/AeEn123/RoExtract/refs/heads/main/packages/arch/PKGBUILD
makepkg -si
```

# Зборка з зыходнікаў

Для зборкі з зыходнікаў патрабуецца cargo, [які можна ўсталяваць праз rustup.](https://rustup.rs)

## 1. Клануйце рэпазіторый
```bash
git clone https://github.com/AeEn123/RoExtract
cd RoExtract
```

## 2. Збярыце з дапамогай cargo, патрэбная каманда залежыць ад сцэнарыя выкарыстання
Калі вы хочаце рэлізную зборку, якая хутка працуе, але доўга кампілюецца (рэкамендуецца для звычайнага выкарыстання)
```bash
cargo build --release
```

Калі вы хочаце перадрэлізную зборку, якая павольна працуе, але хутка кампілюецца (рэкамендуецца для распрацоўкі)
```bash
cargo build
```
Дачакайцеся заканчэння зборкі залежнасцей і самой праграмы. Пасля гэтага, вы знойдзеце зборку ў тэчцы `target`.
